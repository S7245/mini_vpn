# VPS 安全组推荐配置

本文基于以下两份服务端安装文档整理：

- `knife8-singbox-server-setup.md`：VLESS + REALITY，服务端监听 `TCP 443`
- `13-tuic-tcp-connect.md`：TUIC v5，服务端监听 `UDP 8443`

目标是按最小暴露原则配置 VPS 安全组，减少被公网扫描器和网络蠕虫侵入的风险。

## 推荐入站规则

默认策略：拒绝所有入站，只显式放行必要端口。

| 优先级 | 协议 | 端口 | 来源 | 用途 | 建议 |
|---:|---|---:|---|---|---|
| 1 | TCP | 22 | 你的固定管理 IP `/32` 或办公 VPN 网段 | SSH 管理 | 不要对 `0.0.0.0/0` 开放 |
| 2 | TCP | 443 | 固定客户端 IP `/32`；无法固定时才用 `0.0.0.0/0` | REALITY/VLESS 服务 | 必要业务入口 |
| 3 | UDP | 8443 | 固定客户端 IP `/32`；无法固定时才用 `0.0.0.0/0` | TUIC 服务 | 必要业务入口 |
| 4 | ICMP | 无端口 | 你的管理 IP 或监控源 | ping/网络诊断 | 可选，不影响 VPN 服务 |
| 99 | ALL | ALL | ALL | 其他入站流量 | 拒绝 |

不要额外开放这些端口，除非有明确用途：

- `TCP 80`
- `TCP 8443`
- `UDP 443`
- 数据库端口，如 `3306`、`5432`、`6379`
- Docker、面板、监控、调试服务端口

## 推荐出站规则

默认策略可以按云厂商能力选择：

- 简化配置：允许全部出站，重点收紧入站。
- 更严格配置：只放行下表中的必要出站。

| 协议 | 端口 | 目标 | 用途 | 建议 |
|---|---:|---|---|---|
| TCP | 443 | REALITY 借用站 `<DECOY>`，或公网 `0.0.0.0/0` | REALITY 回落/握手借用站访问 | 文档要求 VPS 能访问 `<DECOY>:443` |
| TCP | 80, 443 | 公网 `0.0.0.0/0` | 系统更新、安装软件、普通 HTTPS 出口 | 可保留 |
| UDP/TCP | 53 | 指定 DNS 服务器 | 域名解析 | 推荐指定可信 DNS |
| UDP | 123 | 指定 NTP 服务器，或公网 `0.0.0.0/0` | 时间同步 | 只需要出站，不要开放入站 |

NTP 的目标是时间同步服务器，例如：

- 云厂商内置 NTP
- `pool.ntp.org`
- `time.cloudflare.com`
- `time.google.com`

如果安全组不支持按域名配置目标，可以先将出站 `UDP 123` 目标设为 `0.0.0.0/0`，再在系统里配置可信 NTP 源。

## IPv6 注意事项

文档中的 sing-box 配置使用了：

```json
"listen": "::"
```

这表示服务会监听 IPv6。云安全组需要同步配置 IPv6 入站规则，或者在 VPS 上禁用 IPv6。

不要只收紧 IPv4，而让 IPv6 继续暴露：

- IPv4 入站：`0.0.0.0/0`
- IPv6 入站：`::/0`

如果业务不需要 IPv6，最简单的安全做法是禁用 VPS IPv6 或删除 IPv6 安全组放行规则。

## SSH 加固建议

安全组之外，建议同时做这些基础加固：

- SSH 只允许密钥登录。
- 禁止 root 密码登录。
- SSH 入站只允许你的固定管理 IP。
- 系统和 sing-box 保持更新。
- 不在 VPS 上暴露 Web 管理面板、数据库、Redis、Docker API 等服务。

## 验收检查

在 VPS 上确认监听端口：

```bash
ss -tlnp | grep ':443'
ss -ulnp | grep ':8443'
```

从客户端确认业务可用：

- REALITY 客户端连接 `<VPS_IP>:443`
- TUIC 客户端连接 `<VPS_IP>:8443`

从外部扫描时，理想状态是只看到：

- `TCP 443` 开放
- `UDP 8443` 对 TUIC 客户端可用
- `TCP 22` 只对你的管理 IP 可达

其他端口应不可达。
